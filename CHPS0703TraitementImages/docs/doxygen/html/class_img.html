Description détaillée

Classe Img implémentant le pattern Singleton pour la gestion d'image.

Cette classe modélise une image I : Omega->V au sens du cours, où :

  • Omega ⊂ Z^2 représente le support spatial de l'image (grille cartésienne)
  • V = [0,255]^3 est l'espace des valeurs RGB (quantification 8 bits) L'image peut être vue comme un vecteur VI e R^n dans l'espace vectoriel des images, permettant l'application d'opérateurs linéaires H : E->F et non-linéaires.
Note:\n Sauvegarde de l'image originale pour permettre la composition d'opérateurs
Voir aussi:\n CM02 pour la définition image = vecteur, CM04 pour l'espace vectoriel

Dilatation morphologique (opérateur de rang, supremum local).

Implémente la dilatation : (F oplus B)(x) = sup{F(x-b) | b e B}. Opérateur non-linéaire commutant avec le supremum. Élargit les objets blancs dans le treillis complet (2^Omega, subseteq) pour les images binaires. Duale de l'érosion par l'adjonction fondamentale. Équivalent au filtre maximum.

Paramètres
kernelSizeTaille de l'élément structurant B (carré de côté k)
Note:\n Filtre de rang max : élargit zones lumineuses, atténue taches sombres
Voir aussi:\n CM05 morphologie algébrique, TD#2 Exercice 3 filtres de rang

Égalisation d'histogramme (transformation non-linéaire adaptative).

Érosion morphologique (opérateur de rang, infimum local).

Filtre bilatéral (lissage préservant les contours).

Combine deux types d'information pour un filtrage edge-preserving :

  1. Proximité spatiale : pondération gaussienne sur la distance (comme filtre gaussien)
  2. Similarité d'intensité : pondération gaussienne sur la différence de valeur

Formule : I'(x) = (1/W) * sum{ G_s(||x-y||) * G_r(|I(x)-I(y)|) * I(y) } où G_s est la gaussienne spatiale et G_r la gaussienne de range (intensité).

Réduit le bruit tout en préservant les bords nets. Excellent pour les images où la précision des contours est cruciale.

Paramètres
kernelSizeTaille de la fenêtre (impaire, typiquement 5 ou 7)
sigmaSpatialÉcart-type spatial (proximité géométrique)
sigmaRangeÉcart-type d'intensité (similarité de valeur)
Exceptions:\n std::runtime_error Si paramètres invalides
Note:\n Filtre non-linéaire adaptatif : préserve les discontinuités (contours)
Voir aussi:\n TD#2 Exercice 5 - Filtre bilatéral

Algorithme de détection de contours optimal en plusieurs étapes :

Filtre gaussien (filtre linéaire de lissage pondéré).

Applique une convolution avec un noyau gaussien 2D : G(x,y) = (1/2*pi*sigma^2) * exp(-(x^2+y^2)/(2*sigma^2)). Les pixels proches du centre contribuent plus fortement (pondération gaussienne). Lisse les détails fins tout en préservant mieux les transitions importantes que le filtre moyen. Idéal pour un lissage modéré tout en conservant les contours.

Paramètres
kernelSizeTaille de la fenêtre (impaire, typiquement 5 ou 7)
sigmaÉcart-type de la gaussienne (contrôle l'étendue du lissage)
Exceptions:\n std::runtime_error Si kernelSize pair ou < 1
Note:\n Filtre séparable : peut être décomposé en deux passes 1D pour optimisation
Voir aussi:\n TD#2 Exercice 2 - Filtre gaussien

Filtre médian (filtre de rang robuste au bruit).

Remplace chaque pixel par la médiane des valeurs dans la fenêtre locale. Contrairement aux filtres linéaires, il conserve mieux les contours. Note:\n Préserve les discontinuités (contours) contrairement au filtre moyen

Filtre moyen (filtre linéaire de lissage).

Calcule la moyenne arithmétique des pixels dans une fenêtre locale de taille kernelSize x kernelSize. Opérateur linéaire : I'(x) = (1/|B|) * sum{I(x+b) | b in B} où B est l'élément structurant. Exceptions:\n std::runtime_error Si kernelSize pair ou < 1
Note:\n Filtre passe-bas : atténue les hautes fréquences (détails fins)

Filtre de Prewitt (détection de contours par gradient).

Applique les masques de Prewitt pour détecter les gradients : Gx = [[-1,0,+1],[-1,0,+1],[-1,0,+1]], Gy = [[-1,-1,-1],[0,0,0],[+1,+1,+1]] Similaire à Sobel mais avec pondération uniforme. Calcule G = sqrt(Gx^2 + Gy^2). Détecte les variations abruptes d'intensité pour révéler les contours.

Note:\n Moins sensible au bruit que les opérateurs simples, plus que Sobel
Voir aussi:\n TD#2 Exercice 4 - Filtres différentiels (Prewitt)
